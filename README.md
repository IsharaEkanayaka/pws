# K8s Cluster Manager

A self-service platform for provisioning and managing Kubernetes clusters on Proxmox VE. Combines **Packer**, **Terraform**, **Ansible**, and a **FastAPI** REST API with a web dashboard.

## Architecture

```
┌─────────────────────────────────────────────────┐
│                  Web Dashboard                  │
│             (Single-page HTML/JS)               │
└──────────────────┬──────────────────────────────┘
                   │ HTTP
┌──────────────────▼──────────────────────────────┐
│              FastAPI REST API                    │
│  Auth · Clusters · Environments · Namespaces    │
│  Users · Permissions · Jobs                     │
└──┬────────────────┬─────────────────┬───────────┘
   │                │                 │
   ▼                ▼                 ▼
Terraform       Ansible           Paramiko/SSH
(VM lifecycle)  (K8s setup)       (kubectl)
   │                │                 │
   └────────┬───────┘                 │
            ▼                         ▼
     Proxmox VE API            K8s Control Plane
```

## Quick Start

### 1. Create the VM Template (one-time)

```bash
cd packer
packer init ubuntu-server.pkr.hcl
packer build ubuntu-server.pkr.hcl
```

This creates an Ubuntu 22.04 template (VM ID 990) on Proxmox with cloud-init support.

### 2. Configure Terraform

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Proxmox credentials and desired cluster size
```

Key variables in `terraform.tfvars`:

```hcl
proxmox_api_url          = "https://<proxmox-host>:8006/api2/json"
proxmox_api_token_id     = "user@pam!token-name"
proxmox_api_token_secret = "<secret>"
target_node              = "pve1"
control_plane_count      = 1
worker_count             = 2
```

### 3. Install Python Dependencies

```bash
conda create -n k8s-api python=3.12 -y
conda activate k8s-api
pip install fastapi uvicorn[standard] paramiko
```

### 4. Start the API Server

```bash
uvicorn api.main:app --host 0.0.0.0 --port 8000
```

On first start, a default admin user is created:

| | |
|---|---|
| **Username** | `admin` |
| **Password** | `admin` |

The admin API key is also written to `data/admin_key.txt` for programmatic access.

### 5. Open the Dashboard

Navigate to `http://<host>:8000` and sign in.

## API Reference

Base URL: `/api/v1`

### Authentication

All endpoints require a `Authorization: Bearer <token>` header. Tokens can be:
- **Session tokens** (`ses_...`) — obtained via `POST /auth/login`
- **API keys** (`ak_...`) — assigned to users at creation time

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/auth/login` | POST | None | Sign in with username + password |
| `/auth/logout` | POST | Any | End session |

**Login request:**
```json
{ "username": "admin", "password": "admin" }
```

### Clusters

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/clusters` | POST | Admin | Create cluster (triggers async provisioning) |
| `/clusters` | GET | Viewer+ | List all clusters |
| `/clusters/{id}` | GET | Viewer+ | Get cluster detail |
| `/clusters/{id}` | PATCH | Admin | Update cluster (e.g. assign environment) |
| `/clusters/{id}` | DELETE | Admin | Delete cluster (triggers async destruction) |

**Create request:**
```json
{ "name": "my-cluster", "node_count": 3 }
```

`node_count` is split into 1 control plane + (n-1) workers.

### Environments

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/environments` | POST | Admin | Create environment |
| `/environments` | GET | Viewer+ | List environments |
| `/environments/{id}` | GET | Viewer+ | Get environment detail |
| `/environments/{id}` | DELETE | Admin | Delete (must have no active clusters) |

### Namespaces

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/clusters/{id}/namespaces` | POST | Team Lead+ | Create namespace (runs kubectl) |
| `/clusters/{id}/namespaces` | GET | Viewer+ | List namespaces |
| `/clusters/{id}/namespaces/{ns_id}` | DELETE | Team Lead+ | Delete namespace |

### Jobs

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/jobs/{id}` | GET | Viewer+ | Get job status and error details |

### Users & Permissions

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/users/me` | GET | Any | Current user info |
| `/users` | POST | Admin | Create user (returns API key once) |
| `/users` | GET | Admin | List users |
| `/users/{id}` | DELETE | Admin | Deactivate user |
| `/users/{id}/permissions` | GET | Admin | List user permissions |
| `/permissions` | POST | Admin | Grant permission |
| `/permissions/{id}` | DELETE | Admin | Revoke permission |

## RBAC Roles

| Role | Level | Capabilities |
|------|-------|-------------|
| `admin` | 4 | Full access. Create/delete clusters, users, environments. |
| `team_lead` | 3 | Manage namespaces, write access to assigned resources. |
| `developer` | 2 | Read access to assigned resources. |
| `viewer` | 1 | Read-only access to assigned resources. |

Admins bypass all resource-level permission checks. Other roles require explicit permission grants per resource.

## Provisioning Flow

When a cluster is created via the API:

1. **IP allocation** — reserves a contiguous block of IPs from the pool (starting at `.201`)
2. **Workspace setup** — creates `workspaces/<cluster_id>/terraform/` with symlinked `.tf` files and a per-cluster `terraform.tfvars`
3. **Terraform apply** — clones the VM template, configures static IPs via cloud-init
4. **Ansible playbook** — runs against the auto-generated inventory:
   - `common` role: sets hostnames, disables swap, installs containerd + kubeadm
   - `control_plane` role: `kubeadm init`, installs Flannel CNI, generates join token
   - `worker` role: joins workers to the cluster, labels them
5. **Status update** — cluster status moves from `creating` → `running` (or `failed`)

All provisioning runs in background threads. The UI polls every 5 seconds while clusters are in transitional states.

## Database

SQLite at `data/api.db`. Tables:

- `clusters` — cluster metadata, status, IP allocation
- `environments` — logical groupings for clusters
- `jobs` — async provisioning/destruction job tracking
- `ip_allocations` — tracks which IP ranges are in use
- `namespaces` — cluster namespace records
- `users` — credentials (password hash + API key hash)
- `sessions` — active login sessions
- `permissions` — per-resource access grants

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Cluster stuck in `creating` | Check job status via Jobs page. Look at API server logs for Terraform/Ansible errors. |
| SSH "Permission denied" to VMs | Default VM credentials are `ubuntu`/`ubuntu`. Ensure Terraform `ssh_user`/`ssh_password` match. |
| Stale SSH host keys after VM recreation | Run `ssh-keygen -R 10.40.19.<ip>` for affected IPs. |
| Namespace creation fails | Cluster must be in `running` status. API uses Paramiko SSH to the control plane — verify connectivity. |
| Apt lock errors during provisioning | The `common` role waits for apt locks and disables unattended-upgrades. If it persists, SSH into the VM and check `fuser /var/lib/dpkg/lock-frontend`. |
| Terraform clone timeout | Proxmox storage I/O may be slow. VMs may get stuck in `lock: clone` state — use the Proxmox UI to unlock/destroy them. |
| Port 8000 already in use | `fuser -k 8000/tcp` to free the port before restarting. |
